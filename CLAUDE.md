## Project Overview

Master thesis project predicting how football players' performance qualities change after within-league transfers, using **Twelve Football (Wyscout)** player/team data and **Transfermarkt** transfer records. The core approach is per-position OLS regression: given a player's pre-transfer qualities and the tactical styles of origin/destination teams, predict post-transfer player qualities.

## Data Setup

Raw data lives outside the repo in `../thesis_data/`:
- `raw_data/` — source parquets (Teams_stats, Transfers, Wyscout)
- `processed_data/` — auto-generated by notebooks
  - `master_dataset/` — full 262K-row master parquet
  - `practice_model_dataset/` — earlier practice iterations
  - `thesis_model_dataset/active/` — active modelling data (`within_league_transfers_v5.parquet`, `team_qualities.parquet`)

Data files are never committed. The repo contains only notebooks, quality definitions, and scripts.

## Architecture

### Practice models (`practice_model/`)

- **v_1**: 8 leagues, ~7,400 transfers. Lasso wins (R²=0.369). XGBoost overfits.
- **v_2**: 228 competitions, ~78K rows. Lasso/Ridge still win (~R²≈0.31). XGBoost still overfits.

### Preprocessing pipeline (`thesis_model/python_notebooks/preprocessing/`)

Notebooks 01–08 process the master dataset (262K rows, 539 cols) into the modelling-ready `within_league_transfers_v5.parquet` (~18K within-league transfers, 120 cols).

### Models (`thesis_model/python_notebooks/`)

Three model iterations exist:

- **`models/`** — v1: 6 separate notebooks (naive, all pre-Q, pre-Q+teams, delta team, pre-Q+delta, +controls). One per model, midfielders only.
- **`thesis_models_v2/`** — v2: Consolidated into 2 notebooks (dataset overview + model comparison). Discovered regression-to-mean inflation in delta models.
- **`thesis_models_v3/`** — v3 (current): 2 notebooks, 5 models. Fair comparison — all models evaluated on `to_Q` (delta models reconstruct `to_Q = from_Q + predicted_delta`). Dropped controls model (marginal gains).

### v3 Models (current)

| # | Name | Features | Target |
|---|------|----------|--------|
| 1 | Naive Baseline | `from_Q_i` (1) | `to_Q_i` |
| 2 | Player Profile | `from_Q_1..17` (17) | `to_Q_i` |
| 3 | Player + Team Context | `from_Q_1..17` + `from_team_1..7` + `to_team_1..7` (31) | `to_Q_i` |
| 4 | Tactical Shift | `delta_team_1..7` (7) | `to_Q_i` |
| 5 | Player + Tactical Shift | `from_Q_1..17` + `delta_team_1..7` (24) | `delta_Q_i` → `to_Q_i` |

**Key finding**: Model 3 (Player + Team Context) wins 13/17 qualities. No single best model — best model varies by quality.

### Key concepts

**Twelve Quality Scores** (20 per player, from + to): Composite indices from position-adjusted z-scores. 17 used for midfielders (3 excluded: Chance prevention, Territorial dominance, Poaching — >50% null).

**Team style qualities** (7): Defence, Defensive Transition, Attacking Transition, Attack, Penetration, Chance Creation, Outcome. Defined in `twelve_qualities/team_qualities.txt`.

**Positions** (5 outfield): Central Defender, Full Back, Midfielder, Winger, Striker. Current models focus on Midfielders.

**Train/test split**: Temporal — train 2018–2023, test 2024.
